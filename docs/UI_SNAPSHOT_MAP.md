# UI_SNAPSHOT_MAP

说明：本清单用于人工验收与后续 E2E。每个点位均要求 URL 可直接打开、关键元素可见、文案一致。

## P0 登录页（/login）
- 区域：页面主体表单区域
- 关键元素：邮箱输入、密码输入、登录按钮、去注册入口
- 文案：`登录`、`没有账号？去注册`

## P1 Dashboard（/）
1. Header 导航
- 关键元素：`仪表盘`、`搜索`、`订阅与投递`、`Admin`（仅管理员可见）
- 当前页高亮：`/` 时高亮 `仪表盘`

2. KPI 区（近30天汇总）
- 关键元素：3-5 个 KPI 卡片、每卡有 `查看列表`（后续 Phase 补齐）

3. LRI Top 区块
- 关键元素：表头、至少 5 行、`查看全部`（后续 Phase 补齐）
- 钻取：第 1 行可点击跳转 `/search`

4. 高风险生命周期证 Top10
- 关键元素：Top10 列表、`查看全部`（后续 Phase 补齐）
- 钻取：第 1 行跳转 `/search?registration_no=...`

5. 高竞争格局证 Top10
- 关键元素：Top10 列表、`查看全部`（后续 Phase 补齐）
- 钻取：第 1 行跳转 `/search?q=...`

6. 趋势图（近30天）
- 关键元素：趋势图容器、可扩展 Tab（后续 Phase 补齐）

7. LRI Map（方法学×分类）
- 关键元素：方法学与分类维度、`查看该赛道产品`按钮（后续 Phase 补齐）

## P2 Search（/search）
1. 筛选区
- 关键元素：关键词、企业、注册证号、状态、排序、`includePending`（对应 include_unverified）

2. Saved Views 区
- 关键元素：视图下拉、`保存视图`、管理入口（后续 Phase 补齐）

3. 结果列表首屏
- 关键元素：产品名、企业、注册证号、状态、失效日期、UDI-DI、Badge

4. 导出 CSV
- Free：点击 `导出 CSV` 展示统一 ProGate，不触发下载
- Pro：点击 `导出 CSV` 触发原下载逻辑

## P3 Detail（/products/[id] 或 /registrations/[no]）
1. 概览区
- 关键元素：产品名/企业/注册证号（可复制）/状态/关键日期

2. 结构化字段区
- 关键元素：至少 3 组折叠字段（后续 Phase 补齐）

3. 证据与变更区
- 有数据：证据列表 + 变更时间线
- 无数据：显示 Empty state `暂无数据`

4. 返回路径
- 关键元素：返回按钮或浏览器后退，返回 Search 且筛选不丢

## P4 Admin（/admin）
1. Admin 导航
- 关键元素：数据质量/风险运营/系统 分组入口

2. 队列卡片
- 关键元素：待处理文档、UDI 待映射、冲突待裁决、LRI 高风险

3. 原因码 TOP
- 关键元素：reason_code 列表 + 数量

## P5 Admin 队列页（/admin/queue/*）
- 顶部标题
- 过滤器（至少 1 个）
- 列表 + 分页/加载更多
- 批量操作按钮（已处理/忽略）

## P6 原因码详情（/admin/reasons/[code]）
- 关键元素：reason_code、统计数、10 条样本
- 关键元素：`复制工单文本` 按钮
- 复制内容需包含：reason_code、sample_ids(>=3)、复现路径、建议修复点
